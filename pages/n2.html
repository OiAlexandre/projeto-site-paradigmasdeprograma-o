Dominio de Programa


Os computadores possuem propósitos em uma infinidade de áreas, como a área cientifica, comercial, aplicações de I.A., linguagens script, dentre outros...


Primitivas


São os comandos básicos que uma linguagem de programação fornece para o programador. Elas são usadas para realizar tarefas simples, como atribuição de valores a váriaveis, entrada e saída de dados e operações matemáticas lógicas.


Criterios de Avaliação de uma Linguagem


  1. Legibilidade;
    • Capacidade de escrita;
  2. Simplicidade e Ortogonalidade;
    • Suporte para abstração;
  3. Expressividade;
  4. Confiabilidade;
  5. Custo;

Legibilidade


A legibilidade pode ser avaliada por meio das seguintes formas:


Simplicidade Global: Refere-se ao príncipio de que um sistema ou programa deve ser implementado e projetado de forma simples e fácil de entender.


Ortogonalidade: Significa que as funcionalidades do sistema são independentes umas das outras, de modo que uma mudança em uma parte do sistema não afeta as outras


Instruções de Controle: Se referem as estruturas de controle em um programa, como condições (if, else), loops (for, while) e instruções de desvio.


Tipos de Dados e Estruturas: Referem-se aos diferentes tipos de valores que podem ser armazenados em uma váriavel ou estrutura de dados, como números inteiro, números de ponto flutuante, string, etc...


Sintaxe: São regras gramaticais e a estrutura de um código em uma linguagem de programação expecífica, cada linguagem tem sua propría sintaxe


Capacidade de Escrita


A capacidade de escrita, ou writability, é a medida de quão facilmente uma linguagem pode ser utilizada para criar programas para um domínio de problema escolhido. Pode ser considerada com base nas características de:


Simplicidade e ortogonalidade: Caso uma linguagem de programação tenha um grande número de diferentes construções, alguns programadores podem não estar familiarizados com todas elas. Um número menor de construções primitivas e um conjunto consistente de regras para combiná-las (ortogonalidade) é muito melhor do que existir um grande número de primitivas.


Suporte para abstração: A abstração é a capacidade de definir e, depois, de usar estruturas ou operações complexas de uma maneira que permita ignorar muitos dos detalhes de implementação. Isto é possível, por exemplo, por meio da implementação de métodos. Se criarmos métodos para manipular uma estrutura de dados do tipo pilha (inserir dados na pilha – empilhar, retirar dados da pilha – desempilhar e assim por diante) outros desenvolvedores poderão, posteriormente, manipular uma pilha apenas chamando estes métodos. Uma estrutura de pilha é uma estrutura de dados que armazena elementos em uma ordem específica, chamada de LIFO (Last In, First Out). Isso significa que o último elemento a ser inserido na pilha é o primeiro a ser removido.


Expressividade: A expressividade é a capacidade de que operadores muito poderosos permitam que uma grande quantidade de computação seja realizada com um programa muito pequeno.


Confiabilidade: Um programa é considerado confiável se ele se comportar de acordo com as suas especificações sob todas as condições.
